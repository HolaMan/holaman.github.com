View & Activity

View與Activity的關係以及相關的程式範例

說明

Activity是一個流程的單位概念，可以指定一個View來呈現給使用者看。

View是視覺元件的單位，他可以又繼續擁有許多子元件 (依然是View)，形成一顆視覺樹 (View Collection)。

Activity可以藉由setContentView來將這顆視覺樹與Activity作連結。

另外如果在Activity想要找出視覺樹裡特定的元件，可以用findViewById，傳入元件的id來尋找視覺元件。

程式範例

in source code of Activity (src)

public class ToastActivity extends Activity {
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_toast);        
        
        Button button4 = (Button)this.findViewById(R.id.button4);
        button4.setOnClickListener(new OnClickListener() {
    @Override
      public void onClick(View v) {
        Toast.makeText( ToastActivity.this, "按鈕被按了....", Toast.LENGTH_LONG).show();
      }});
    }
}
